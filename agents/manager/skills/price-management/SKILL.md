# 💰 房价管理 (Price Management)

## 触发条件

当用户提到：房价、价格、调价、涨价、降价、改价、标准间多少钱、豪华房价格、套房价格。

## 工作流程

### 展示当前价格

首先展示当前房价一览表，然后给出操作选项：

```
💰 当前房价一览

| 房型 | 门市价 |
|------|--------|
| 标准间 | ¥388 |
| 豪华房 | ¥560 |
| 套房 | ¥1,280 |
| 总统套房 | ¥3,880 |

请选择操作：
1️⃣ 修改房价
2️⃣ 设置特殊日期价格
3️⃣ 查看价格变更历史
4️⃣ 返回
```

### 操作 1: 修改房价

1. **权限检查** — 需要 `price.modify` 权限（老板、总经理）
2. 请用户输入房型和新价格，格式 `标准间 428`
3. 计算涨跌幅
4. **大幅调价预警** — 涨跌幅 > 20% 时，需要老板权限确认
5. **二次确认**:
```
🔔 房价调整确认
📈 标准间: ¥388 → ¥428 (+10.3%)
确认执行？
```
6. 确认后更新价格，通知前台 Agent

### 操作 2: 设置特殊日期价格

格式：`日期 房型 价格 标签`，例如 `2026-03-01 标准间 468 周末`

### 操作 3: 查看变更历史

列出最近 10 条价格变更记录。

## 权限规则

| 角色 | 权限 |
|------|------|
| 老板 (boss) | ✅ 查看 + 修改（含大幅调价） |
| 总经理 (general_manager) | ✅ 查看 + 修改 |
| 运营经理 (operations_manager) | 👁️ 仅查看 |
| 部门主管 (department_head) | 👁️ 仅查看 |
| 普通员工 (staff) | ❌ 无权限 |

## 大幅调价规则

价格调整幅度超过 **20%** 时必须由老板角色执行。其他人尝试时回复：
```
⚠️ 大幅调价预警

标准间: ¥388 → ¥500 (+28.9%)

价格调整幅度超过 20%，需要老板权限确认。
```

## 注意事项

- 修改价格后提示"已通知前台 Agent 更新"
- 模拟模式下价格变更仅在当前会话中生效
- 默认用户 ID 为 U001（张老板），拥有全部权限
